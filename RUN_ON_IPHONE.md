# 아이폰에서 미리보기 (컴퓨터 없이, 태블릿 + 아이폰)

컴퓨터 없이 **Expo Snack**(브라우저)으로 이 앱을 아이폰에서 바로 띄워보는 방법입니다.
화면·기록 저장·적정가 판정 흐름을 확인할 수 있습니다.
(카메라 촬영 + OCR은 이 단계에선 동작하지 않습니다 — 정식 빌드 단계에서 활성화.)

---

## 사전 준비 (한 번만)

### 1. 저장소를 공개로 전환
Snack은 **공개 저장소만** 가져올 수 있습니다. 태블릿 브라우저에서:
1. `https://github.com/NanpaKim/my-wine` 접속 → **Settings**
2. 맨 아래 **Danger Zone** → **Change repository visibility** → **Make public** 선택
3. 확인 문구 입력 후 전환

> 공개하면 코드가 인터넷에 공개됩니다. 개인 와인 기록 앱이라 보통 문제는 없지만,
> 원치 않으면 README의 "방법 B(클라우드 개발환경)"로 진행하세요.

### 2. 아이폰에 Expo Go 설치
App Store에서 **"Expo Go"** 검색 → 설치 (무료).

---

## 아이폰에서 띄우기

1. **태블릿 브라우저**에서 `https://snack.expo.dev` 접속
2. 왼쪽 위 메뉴에서 **Import** (또는 "Import git repository") 선택
3. 저장소 주소 붙여넣기:
   ```
   https://github.com/NanpaKim/my-wine
   ```
   → **Import** 누르기 (코드는 `main` 브랜치에 있습니다)
4. 불러와지면, 오른쪽 패널에서 **My Device** 탭 선택 → **QR 코드**가 나타남
5. **아이폰 기본 카메라 앱**으로 그 QR을 비추면 → "Expo Go에서 열기" 배너 탭
6. 잠시 빌드 후, 아이폰에서 **my-wine** 앱이 실행됩니다 🎉

> 로그인을 요구하면, Snack과 아이폰 Expo Go **둘 다 같은 Expo 계정**으로 무료 가입/로그인하세요.

## 앱에서 해볼 것 (현재 동작하는 흐름)

1. 홈 화면 우측 하단 **＋ 라벨 촬영** 탭
2. **직접 입력으로 기록하기** 탭 (카메라는 다음 단계)
3. 구입처(소매/식당)·가격·음식·평점·날짜 입력 → **저장**
4. 홈 화면에 기록이 뜨고, **적정가 판정**(적절/비쌈/저렴)이 표시됩니다
   - 단, 현지 시세 연동(스텁) 전이라 지금은 대부분 "판단 불가"로 나옵니다.
     시세 연동은 DESIGN.md §4 / §7의 다음 단계 작업입니다.

## 잘 안 될 때

| 증상 | 해결 |
|---|---|
| 가져오기 에러 / "invalid project" | 저장소가 **공개**인지, 주소가 정확한지 확인 |
| 빈 화면만 보임 | Snack이 `main`을 가져왔는지 확인 (코드는 main에 있음) |
| **SDK 버전 안 맞음** 경고 | 우리 앱은 최신 SDK라 Snack이 아직 미지원일 수 있음 → 알려주시면 SDK를 낮춰 드리거나, 방법 B로 전환 |
| Expo Go에서 안 열림 | 아이폰과 태블릿이 인터넷에 연결돼 있는지, Expo Go 최신 버전인지 확인 |

막히면 어느 단계에서 어떤 메시지가 나왔는지 알려주세요. 이어서 도와드리겠습니다.
